**Imports:**
```javascript
import { state } from './state.js';
import { parseXML } from '../core/parser.js';
import { buildTree, sortChildren } from '../core/tree.js';
import { isReleased } from '../core/merge.js';

import { importArtifact, validateArtifact, suggestRevision, suggestJobNumber } from '../core/artifact.js';
```

**Section 1: DOM Element References**
Get all DOM elements by ID (following hierarchy.js pattern). Include: ifpSourceZone, ifpSourceFile, ifpSourceInfo, ifpSourceName, ifpSourceMeta, ifpPriorZone, ifpPriorFile, ifpPriorInfo, ifpPriorName, ifpPriorMeta, ifpPriorContainer, ifpRev0Toggle, ifpMessage, ifpValidationErrors, ifpMergeBtn, ifpResetBtn, ifpResults, ifpTreeBody, ifpExpandAllBtn, ifpCollapseAllBtn, ifpHideWipToggle, ifpHideGraftedToggle, ifpPassedCount, ifpGraftedCount, ifpPlaceholderCount, ifpSummaryStats, ifpControlBar, ifpExportBtn, ifpRevisionInput, ifpJobNumber, and any assembly info display elements.

**Section 2: REV0 Toggle**
When checkbox changes:
- If checked: set `state.ifpIsRev0 = true`, add class `ifp-prior-disabled` to `ifpPriorContainer`, clear any loaded B(n-1) state.
- If unchecked: set `state.ifpIsRev0 = false`, remove class `ifp-prior-disabled` from `ifpPriorContainer`.
- After toggle change, call `updateMergeReadiness()` to re-evaluate whether merge button should be enabled.

**Section 3: X(n) Source Export Upload**
Follow hierarchy.js upload pattern exactly (click, dragover, dragleave, drop, change handlers).
- Accept only `.xml` files.
- On file load: decode as UTF-8, call `parseXML(xmlText)` to get rows, store in `state.ifpSourceData`, store filename in `state.ifpSourceFilename`.
- Build tree: `state.ifpSourceTree = buildTree(state.ifpSourceData)` then `sortChildren(state.ifpSourceTree)`.
- After tree is built, immediately run validation: `state.ifpValidationResult = validateBOM(state.ifpSourceTree)`.
- Display the tree using `displayIfpTree(state.ifpSourceTree)`.
- Show validation errors if any (see Section 6).
- Call `updateMergeReadiness()`.
- Show success message with row count.

**Section 4: B(n-1) Prior Artifact Upload**
Follow same upload pattern but for JSON files.
- Accept only `.json` files.
- On file load: read as text (UTF-8), call `importArtifact(jsonString)` to parse.
- Store artifact in `state.ifpPriorArtifact`, filename in `state.ifpPriorFilename`.
- Run async validation: `const result = await validateArtifact(state.ifpPriorArtifact, { expectedGA: state.ifpSourceTree?.partNumber })`.
  - If `result.valid === false` (hash mismatch): show error message, reject the file, clear state.ifpPriorArtifact.
  - If warnings exist: show them in the message area as informational (not blocking).
- Auto-suggest revision and job number if source tree exists:
  - `ifpRevisionInput.value = suggestRevision(state.ifpPriorArtifact)`
  - `ifpJobNumber.value = suggestJobNumber(state.ifpPriorArtifact, state.ifpSourceTree?.partNumber)`
- Call `updateMergeReadiness()`.
- Show success message with artifact info (revision, job number).

**Section 5: State-Aware Tree Rendering**
Create `displayIfpTree(root)` function that renders the BOM hierarchy into `ifpTreeBody`, adapted from hierarchy.js `renderTreeNode` but with these modifications:

- **Simplified columns:** Part Number (tree cell with indentation), Qty, Description, Revision, State (pill), Source (indicator).
- **State pill column:** For EVERY node (assemblies AND parts per user decision), create a `<span class="state-pill released">` or `<span class="state-pill wip">` based on `isReleased(node.state)`. Pill text: "Released" or "WIP".
- **Source column:** Show `node._source` if present (will be populated after merge in plan 02). For pre-merge display, leave empty or show "current".
- **Default expansion:** First level expanded — the root node's direct children are visible (not collapsed), but deeper levels start collapsed. Implementation: when rendering, if `depth === 0` (root) or `depth === 1` (direct children), render as expanded. Deeper levels render as collapsed.
  - Root row: visible, toggle shows "-" (expanded)
  - Depth 1 children: visible (not collapsed class), their toggles show "+" (collapsed)
  - Depth 2+: have class `collapsed`
- **Tree lines:** Copy the tree-line rendering from hierarchy.js (linesContainer, vertical lines, horizontal lines) exactly. The tree-cell class and CSS already handle this.
- **Toggle behavior:** Same `toggleChildren` function as hierarchy.js.
- Show the results section, assembly info, and control bar when tree is displayed.

**Section 6: Validation Error Display**
Create `displayValidationErrors(validationResult)`:
- If `validationResult.valid === true`: hide the banner, enable merge button (if other conditions met).
- If `validationResult.valid === false`: show `ifpValidationErrors` with class `show`.
  - Banner title: "Validation Errors — Merge Blocked"
  - List each error from `validationResult.errors` using the `.error-item` class. Display the error `message` as-is (it already includes full path, rule, and fix action per Phase 13 design).
  - Disable merge button.

**Section 7: Update Merge Readiness**
Create `updateMergeReadiness()`:
- Merge button enabled when ALL of:
  1. X(n) source tree exists (`state.ifpSourceTree !== null`)
  2. Validation passed (`state.ifpValidationResult?.valid === true`)
  3. Either REV0 mode (`state.ifpIsRev0`) OR prior artifact loaded (`state.ifpPriorArtifact !== null`)
- Also auto-suggest revision: if REV0, set revision to 0 and job number to "1J" + root PN.

**Section 8: Expand/Collapse All**
Copy expand/collapse logic from hierarchy.js but target `#ifpTreeBody`:
- Expand All: remove `.collapsed` from all `.child-row`, update toggles to `-` and `.expanded`.
- Collapse All: add `.collapsed` to all `.child-row`, update toggles to `+` and `.collapsed`.

**Section 9: Hide WIP Content Toggle**
`ifpHideWipToggle` click handler:
- Toggle active state (add/remove `active` class, matching filter-btn pattern).
- When active ("Hide WIP"): for each row in `#ifpTreeBody`, if the row represents a WIP assembly (`data-wip-assembly="true"`), hide all its descendant rows by adding class `wip-hidden`. The WIP assembly row itself stays visible (per user decision: keeps WIP assembly visible if parent is Released).
- When inactive ("Show WIP"): remove `wip-hidden` class from all rows.
- Add CSS: `.wip-hidden { display: none; }` — add this to styles.css in Task 1's CSS additions.
- Implementation: during tree rendering, tag each row with `data-wip-assembly="true"` if it represents a WIP assembly (isAssembly && !isReleased). Also tag each row with `data-depth` (already done in hierarchy pattern). The hide logic walks the DOM: for each WIP assembly row, hide all subsequent siblings until reaching a row at same or lesser depth.

**Section 10: Reset/Start Over**
